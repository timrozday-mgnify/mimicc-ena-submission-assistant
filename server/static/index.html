Workflow

1. Create a study → 2. Enter sample metadata in Samples (DataHarmonizer) → prepare & submit → 3. In Reads, assign FASTQ files to samples and submit via webin-cli → 4. Manage everything in Records. The TEST/PROD toggle (top right) selects the ENA service for every action.

Schema library

Saved/imported LinkML schemas, persisted alongside the other backend files. Pick one on the Samples or Reads tab to back that grid, or supply your own from disk below.

Build a new schema

Import fields from bundled ENA sample-checklist XML, ENA/SRA XSD, and/or an existing saved schema. Selected sources are merged (earlier selections win on conflicting fields) and loaded into the editor below.

Edit / preview schema

The dataharmonizer-template-builder sidecar isn't reachable — start it (the dhtb service in docker-compose.yml) to edit schemas. You can still select and save schemas without it.
